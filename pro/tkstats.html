tkstats.info

Loveley project of mine, I have this wierd love to investigate software and take them apart. While I was trying out the game myself I felt this constant annoyance of figuring out what items I wanted to use, comparing A, B and C in-game. There is no real wiki to turn to where you can browse through the items either.

It has been a really fun project to work with, with diffrent things to find solutions for.

Data gathring

First I inject a jump to my own lua code, read all the item IDs from their locale files and then I step through all the IDs to call on GetDataFromIni_lua where it will return a table with related data. Then dumped into CSV files.

Data parsing

This part of the project is to make all necessary alteration of the current csv files and form it into appropriate format(json) for our front-end. I decided to use python for this.

The images wasn't included in the luatable for the items, using "grep -r [SomeItemID] ./" pointed me to the data/ani/. The .ani files includes the itemID and a list of "Frames" associated with the ID, exactly what I needed.

I just parse out the images into my own json file with IDs and its associated images and and make small corrections with the case, for example they mix ".png" and ".PNG". (I guess when they fetch the images it's not case-sensetive?)
Now I can just copy over all the assets from the game folder into my website project and access the images through the associated ID.

Website itself

It's a static website built with Jekyll, it's amazing how far you can go without using a database or any serverside code.

Page generation

I'm using this gem by avillafiorita for the page generation.
With this jekyll plugin I can use the same json files used by the datatables to generate one page for each item, this is achived through by just adding the path to the json data in the _config.yml and we can access all the jsonobject values through liquid tags. Like so~{{ page.ItemName }}

DataTables

For the searchable and sortable datatable I'm using the DataTables plugin for JQuery. The JQuery datatabe makes it really easy for the data population, to use our generated ajax data we only need to add the path to our json file.
